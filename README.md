# 🌱 Smart Farming Advisor

AI-powered crop recommendation and profit estimation system.  
Enter soil + climate data → get recommended crop, yield forecast, market price, cost, and profit.

---

## Architecture

```
smart-farming-advisor/
├── backend/          FastAPI (Python)  — ML models, prediction API
│   ├── data/         Place the 3 CSV files here
│   ├── models/       Auto-generated by train_models.py
│   ├── app/
│   │   ├── main.py
│   │   ├── schemas.py
│   │   └── predictor.py
│   ├── train_models.py
│   ├── requirements.txt
│   └── tests/

    └── src/

        ├── api/predict.ts
        ├── components/

```

## Datasets

Place these 3 CSV files in `backend/data/` before training:

| File | Purpose |
|------|---------|
| `Crop_recommendation.csv` | Train crop recommendation model (RandomForestClassifier) |
| `crop_yield.csv` | Train yield model + per-crop yield lookup (RandomForestRegressor) |
| `Agriculture_price_dataset.csv` | Build crop → Modal_Price mapping |

## Local Setup

### 1 — Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2 — Train models

```bash
# Make sure the 3 CSVs are in backend/data/
python train_models.py
```

Expected output:
```
Crop model accuracy : 99.32%
Yield model R²      : 0.9375
price_mapping.joblib: 5 commodities
```

### 3 — Start backend

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

- Swagger docs: http://127.0.0.1:8000/docs  
- Health check: http://127.0.0.1:8000/health

### 4 — Frontend

```bash
cd frontend
cp .env.example .env          # edit VITE_API_BASE_URL if needed
npm install
npm run dev
```

Open: http://localhost:5173

## API

### `POST /predict`

**Request body:**

```json
{
  "N": 90,
  "P": 42,
  "K": 43,
  "temperature": 20.9,
  "humidity": 82.0,
  "ph": 6.5,
  "rainfall": 202.9,
  "area": 2.5,
  "fertilizer": 100.0,
  "pesticide": 10.0
}
```

**Response:**

```json
{
  "crop": "rice",
  "yield_per_acre": 8.98,
  "total_yield": 22.45,
  "price_per_quintal": 3582.89,
  "total_cost": 125000.0,
  "estimated_profit": -44564.12,
  "units": {
    "yield_per_acre": "quintal/acre",
    "total_yield": "quintal",
    "price_per_quintal": "₹/quintal",
    "total_cost": "₹",
    "estimated_profit": "₹"
  }
}
```

**Units:**

| Field | Unit |
|-------|------|
| temperature | °C |
| humidity | % |
| rainfall | mm |
| area | acre |
| N / P / K | dataset soil value |
| fertilizer / pesticide | dataset unit |
| yield_per_acre | quintal/acre |
| total_yield | quintal |
| price_per_quintal | ₹/quintal |
| total_cost | ₹ |
| estimated_profit | ₹ |

**Profit formula:**  
`estimated_profit = (total_yield × price_per_quintal) − total_cost`

## Deployment

### ⚠️ GitHub Pages cannot run the Python backend

GitHub Pages is a static file host — it only serves HTML/CSS/JS.  
The FastAPI backend **requires a Python server**. Deploy it separately.

### Backend → Render

1. Push code to GitHub
2. Render → New Web Service → connect your repo
3. Settings:
   - **Root Directory:** `backend`
   - **Build Command:** `pip install -r requirements.txt && python train_models.py`
   - **Start Command:** `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
4. Add environment variable:  
   `CORS_ORIGINS` = `https://your-app.vercel.app`

> **Note:** Add the 3 CSV files to the repo (or upload via Render disk) so `train_models.py` can run during build.

### Frontend → Vercel

1. Vercel → New Project → import your repo
2. Settings:
   - **Framework Preset:** Vite
   - **Root Directory:** `frontend`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
3. Add environment variable:  
   `VITE_API_BASE_URL` = `https://your-backend.onrender.com`
